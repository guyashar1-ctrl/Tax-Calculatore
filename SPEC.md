# אפיון מערכת ניהול לקוחות ומשימות — רו"ח

## 1. מטרות ועקרונות

### מה המערכת
מערכת ניהול פרקטיקה אישית של רו"ח, שמאחדת:
- **תיקי לקוחות** (נתונים אישיים, מסמכים, ייפויי כוח) — קיים
- **משימות עבודה** (חדש) — ניהול פעולות שוטפות לכל לקוח
- **אנשי קשר** (חדש) — ניהול בעלי תפקידים הקשורים ללקוחות

### עקרונות מנחים

1. **"אצל מי הכדור"** — כל משימה יודעת בכל רגע איפה היא: אצלי / אצל הלקוח / אצל הרשות / תקועה. המסך הראשי מציג את מה שתורי לטפל בו.
2. **כל משימה קשורה ללקוח** — אין משימות "כלליות" בשלב זה.
3. **FIFO עם קפיצת דד-ליינים** — ברירת מחדל לפי תאריך יצירה, אך משימות עם דד-ליין קרוב קופצות לראש.
4. **מודל מדורג** — בשלב ראשון רק לקוחות עצמאים. הוספת חברות תבוא בהמשך, עם מיגרציה קלה (שדה `type` שיתווסף עם ערך ברירת מחדל `individual`).
5. **קיבוץ אופציונלי** — משימות שאתה רוצה לקבץ לתיק נשארות צמודות עד ביטול. לא חובה, לא דיפולט.

## 2. מודל נתונים

### 2.1 Client (קיים — הרחבה קטנה)

קיימים ~50 שדות של נתוני לקוח עצמאי. תוספת מינימלית:

```typescript
interface Client {
  // ... כל השדות הקיימים ...
  type: 'individual';           // שדה חדש — ברירת מחדל, מכין לחברות בעתיד
  contactSummary?: string;       // טקסט חופשי — הערות כלליות על הלקוח
}
```

**הערה למיגרציה:** כל הלקוחות הקיימים מקבלים אוטומטית `type: 'individual'` בעליה ראשונה.

### 2.2 Contact (חדש)

```typescript
interface Contact {
  id: string;
  fullName: string;              // חובה
  idNumber?: string;             // ת.ז
  phone?: string;
  email?: string;
  address?: string;
  notes?: string;                // טקסט חופשי
  linkedClientId?: string;       // אם איש הקשר עצמו הוא לקוח נפרד — קישור לתיקו
  createdAt: string;
  updatedAt: string;
}
```

**זיהוי אוטומטי של קישור:** אם ת.ז של איש קשר תואמת לת.ז של לקוח קיים, המערכת מציעה אוטומטית `linkedClientId` ומציגה "→ יש לאיש הקשר הזה תיק לקוח". לא מחייב — רק מציע.

### 2.3 ClientContactLink (חדש)

טבלת חיבור רבים-לרבים. תפקיד איש הקשר נשמר כאן, כי **אותו בן אדם יכול להיות "בעל מניות" בלקוח אחד ו"עו"ד" בלקוח אחר**.

```typescript
interface ClientContactLink {
  id: string;
  clientId: string;
  contactId: string;
  role: ContactRole;
  createdAt: string;
}

type ContactRole =
  | 'primary'          // איש קשר ראשי (ברירת מחדל לעצמאי)
  | 'spouse'           // בן/בת זוג
  | 'family'           // בן משפחה (אחר)
  | 'bookkeeper'       // מנהל/ת חשבונות
  | 'lawyer'           // עו"ד
  | 'other_accountant' // רו"ח נוסף
  | 'banker'           // בנקאי
  | 'shareholder'      // בעל מניות (להכנה לחברות)
  | 'other';
```

### 2.4 Task (חדש)

```typescript
interface Task {
  id: string;
  clientId: string;                // חובה — כל משימה שייכת ללקוח
  contactId?: string;              // אופציונלי — איש הקשר הרלוונטי
  caseId?: string;                 // אופציונלי — אם המשימה מקובצת לתיק
  category: TaskCategory;
  title: string;                   // חובה
  description?: string;            // טקסט חופשי
  ballWith: 'me' | 'client' | 'authority' | 'stuck';
  status: 'open' | 'done';         // open כולל גם "in_progress" בשלב ראשון
  priority: 'normal' | 'urgent';
  dueDate?: string;                // YYYY-MM-DD
  createdAt: string;
  updatedAt: string;
  completedAt?: string;
}

type TaskCategory =
  | 'income_tax'           // מס הכנסה
  | 'ni'                   // ביטוח לאומי
  | 'withholdings'         // ניכויים
  | 'vat_report'           // דיווח מע"מ
  | 'withholdings_report'  // דיווח ניכויים
  | 'audit'                // ביקורת
  | 'cutoff'               // חיתוך
  | 'economic_work'        // עבודה כלכלית
  | 'client_onboarding'    // קליטת לקוח
  | 'authority_contact'    // פנייה לרשויות
  | 'other';               // אחר
```

### 2.5 Case (חדש — קיבוץ אופציונלי)

```typescript
interface Case {
  id: string;
  clientId: string;
  title: string;                   // למשל "דוח שנתי 2025"
  createdAt: string;
  closedAt?: string;               // אם סגור
}
```

**התנהגות:** משימה יכולה להיות "חופשית" (`caseId=undefined`) או מקובצת לתיק (`caseId=X`). כפתור "קבץ לתיק" על משימה נבחרת → יוצר `Case` או מוסיף לתיק קיים. כפתור "הוצא מתיק" → `caseId` הופך ל-`undefined`.

### 2.6 אחסון (Storage)

הרחבה של הסכמה הקיימת:

| מפתח | איפה | תוכן |
|---|---|---|
| `crm_clients` | localStorage | Client[] — קיים |
| `crm_representation_requests` | localStorage | קיים |
| `crm_contacts` | localStorage | Contact[] — חדש |
| `crm_client_contact_links` | localStorage | ClientContactLink[] — חדש |
| `crm_tasks` | localStorage | Task[] — חדש |
| `crm_cases` | localStorage | Case[] — חדש |
| `crm_documents_db` | IndexedDB | קיים |

## 3. מסכים

### 3.1 על השולחן שלי (ברירת מחדל בכניסה למערכת)

**הטאב הראשי החדש.** מה שתורי לטפל בו עכשיו.

**פילטר קבוע:** `ballWith = 'me' OR ballWith = 'stuck'`, `status = 'open'`

**מיון:**
1. משימות דחופות (`priority = 'urgent'`) ראשונות
2. משימות עם דד-ליין ב-7 הימים הקרובים (או עבר) — מיון עולה לפי דד-ליין
3. משימות בסטטוס `stuck` — מודגשות בצהוב
4. השאר — FIFO לפי `createdAt`

**wireframe:**

```
┌────────────────────────────────────────────────────────────┐
│  על השולחן שלי                          [+ משימה חדשה]    │
├────────────────────────────────────────────────────────────┤
│  🔴 דחוף (2)                                               │
│  ─────────────────────────────────────                     │
│  ⚠ דוח שנתי 2024 — כהן אברהם         מ"ה   פג 03/05  ❗  │
│  ⚠ ביקורת ניכויים — לוי דוד          ניכ׳  22/04  ❗      │
│                                                            │
│  📅 השבוע (4)                                              │
│  ─────────────────────────────────────                     │
│  ○ לבדוק מסמכי ייעוץ — שמעון סיני    כלכ׳  20/04          │
│  ○ דיווח מע"מ מרץ-אפריל — 12 לקוחות  מע"מ  30/04          │
│  ...                                                       │
│                                                            │
│  🟡 תקועות — צריך חשיבה (1)                                │
│  ─────────────────────────────────────                     │
│  ? פנייה לפ"ש — גולדברג מרים         פנ׳   —              │
```

**פעולות על משימה:** קליק → פתיחת `TaskDetail`. קליק ימני / תפריט → סימון כבוצע, העברת כדור, שינוי דד-ליין, קיבוץ לתיק.

### 3.2 משימות (כל המשימות)

תצוגת על של כל המשימות במערכת, עם פילטרים מלאים.

**פילטרים:**
- לפי סטטוס (פתוחות / סגורות / הכל)
- לפי כדור (אצלי / לקוח / רשות / תקוע / הכל)
- לפי קטגוריה
- לפי לקוח (חיפוש)
- לפי טווח תאריכים

**מיון:** FIFO עם קפיצת דד-ליינים (כמו 3.1).

**קיבוץ תצוגה:** אפשרות להחליף בין "רשימה שטוחה" ל"מקובץ לפי לקוח" או "מקובץ לפי קטגוריה".

### 3.3 לקוחות (קיים — הרחבה קלה)

הטאב הקיים. תוספת: עמודת **"משימות פתוחות"** שמציגה מספר.

```
┌─────────────────────────────────────────────────────────────┐
│  לקוחות                              [+ לקוח חדש]          │
├─────────────────────────────────────────────────────────────┤
│  שם              ת.ז         סוג עצמאי    משימות פתוחות   │
│  ─────────────────────────────────────────────────────     │
│  כהן אברהם      123456789   ע.מורשה      🔴 3            │
│  לוי דוד        234567890   ע.פטור       ○ 1             │
│  ...                                                        │
└─────────────────────────────────────────────────────────────┘
```

### 3.4 תיק לקוח (קיים — הרחבה משמעותית)

**מבנה חדש עם טאבים:**

```
┌────────────────────────────────────────────────────────────┐
│  ← חזור        כהן אברהם (ת.ז 123456789)                  │
│  [פרטים] [משימות] [אנשי קשר] [מסמכים] [ייפוי כוח]         │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  [תוכן הטאב הפעיל]                                        │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

- **פרטים** — הטופס הקיים של ClientForm
- **משימות** — כל המשימות של הלקוח הזה. אפשר להוסיף משימה חדשה מכאן (ה-clientId מולא אוטומטית)
- **אנשי קשר** — אנשי הקשר המקושרים ללקוח + התפקיד שלהם. אפשר להוסיף קיים/חדש
- **מסמכים** — DocumentManager קיים
- **ייפוי כוח** — RepresentationRequest קיים

### 3.5 אנשי קשר (חדש — טאב ראשי)

תצוגה מרכזית של כל האנשים שאתה עובד מולם.

**המסך מאחד שני זרמים:**
1. רשומות `Contact` מה-DB
2. לקוחות מסוג `individual` — מוצגים כאילו היו אנשי קשר (כי הם גם)

**עמודות:** שם, ת.ז, טלפון, מייל, לקוחות קשורים (באיזה תיקים מופיע + תפקיד), האם גם לקוח (✓)

```
┌──────────────────────────────────────────────────────────────────┐
│  אנשי קשר                              [+ איש קשר חדש]  🔍 חיפוש│
├──────────────────────────────────────────────────────────────────┤
│  שם              ת.ז      טלפון       מקושר ללקוחות       גם לקוח│
│  ─────────────────────────────────────────────────────────       │
│  כהן אברהם      123456    050-...     (עצמו)                ✓   │
│  לוי שרה        234567    052-...     ← כהן אברהם (רעיה)    ✗   │
│  ישראלי יוסי    345678    053-...     ← 3 לקוחות             ✓   │
│  ...                                                              │
└──────────────────────────────────────────────────────────────────┘
```

**קליק על איש קשר:** פותח `ContactDetail` — פרטים + רשימת הלקוחות שהוא קשור אליהם (עם התפקיד) + אם הוא גם לקוח, כפתור "עבור לתיק האישי שלו".

### 3.6 ממתינים (תצוגה)

**לא טאב נפרד בהכרח — יכול להיות כפתור פילטר בתוך מסך המשימות.**

## 4. חוקים עסקיים

### 4.1 יצירת לקוח עצמאי — מה קורה ברקע
כשנפתח לקוח עצמאי (`type='individual'`), המערכת **לא** יוצרת אוטומטית `Contact` נפרד — הוא מופיע בטאב אנשי הקשר דרך ה-union. אם מוסיפים לו בן זוג → נוצר `Contact` עבור בן הזוג + `ClientContactLink` עם תפקיד `spouse`.

### 4.2 קישור איש קשר → לקוח
בעת יצירת איש קשר חדש עם ת.ז — המערכת בודקת אם קיים לקוח עם אותה ת.ז. אם כן, מציעה אוטומטית לסמן `linkedClientId`. המשתמש יכול לאשר או לדלג.

### 4.3 מעבר סטטוס כדור
כל פעולה על משימה יכולה לשנות את הכדור. דוגמאות להצעות אוטומטיות (לא כופה):
- יצרתי משימה חדשה → ברירת מחדל `ballWith = 'me'`
- שלחתי בקשה ללקוח → הצעה לסמן `ballWith = 'client'`
- קיבלתי תגובה/מסמכים → הצעה להחזיר ל-`ballWith = 'me'`
- "תקוע" — בחירה ידנית (אני לא יודע איך להתקדם)

### 4.4 קיבוץ לתיק (Case)
- בחירת 2+ משימות → כפתור "קבץ לתיק" → דיאלוג "שם התיק" → כל המשימות מקבלות `caseId`
- בתוך מסך משימות, אפשר להחליף למצב "מקובץ לפי תיק" ואז משימות בתיק יופיעו יחד עם כותרת התיק
- כפתור "הוצא מתיק" על משימה → `caseId = undefined`
- תיק ללא משימות → נסגר אוטומטית

### 4.5 מיון ב"על השולחן"
הלוגיקה המדויקת (סדר):
1. `priority = 'urgent'` וגם `dueDate ≤ היום+3 ימים`
2. `dueDate ≤ היום` (באיחור)
3. `dueDate ≤ היום+7 ימים` (השבוע), מיון עולה לפי `dueDate`
4. `ballWith = 'stuck'`
5. השאר לפי `createdAt` עולה (FIFO)

### 4.6 סגירת משימה
סימון `status = 'done'` → נשמר `completedAt`, המשימה יורדת מהמסך "על השולחן" אך נשארת להיסטוריה במסך "משימות" (פילטר סגורות).

## 5. תוכנית יישום

### שלב 1 — תשתית משימות (MVP מינימלי)
**מטרה:** לקבל מסך "על השולחן" פעיל, משימות קשורות ללקוחות, בסיס לעבוד.

1. הוספת `Task` ו-`Case` ל-`src/types/index.ts`
2. הוספת `useTasks` hook שעוטף localStorage
3. רכיבים חדשים:
   - `TaskForm.tsx` — יצירה/עריכה
   - `TaskList.tsx` — רשימה עם קבוצות ומיון
   - `TaskCard.tsx` — שורה אחת של משימה
   - `MyDesk.tsx` — מסך "על השולחן"
4. עדכון `App.tsx` — הוספת טאב "על השולחן" כדיפולט, טאב "משימות" כרשימה מלאה
5. הוספת טאב "משימות" בתיק הלקוח (`ClientForm` / `ClientDetail`)
6. מיגרציה של `Client` קיימים — הוספת `type: 'individual'` לכל הרשומות

**הישג בסוף השלב:** אתה פותח את האפליקציה ורואה מה לעשות היום. יכול ליצור, לערוך, ולסגור משימות קשורות ללקוחות.

### שלב 2 — אנשי קשר
**מטרה:** פתרון מלא לניהול אנשי קשר + קישורים.

1. הוספת `Contact` ו-`ClientContactLink` ל-`src/types/index.ts`
2. `useContacts` hook
3. רכיבים חדשים:
   - `ContactForm.tsx`
   - `ContactList.tsx` (מסך מרכזי, כולל union עם לקוחות)
   - `ContactDetail.tsx`
   - `ClientContactsTab.tsx` (בתיק הלקוח)
4. לוגיקת זיהוי אוטומטי (ת.ז → לקוח קיים)
5. עדכון טאב "לקוחות" — עמודת משימות פתוחות

**הישג בסוף השלב:** מסך אנשי קשר מרכזי פעיל, כל לקוח חברה בעתיד יוכל לקבל מספר אנשי קשר.

### שלב 3 — ליטוש, פילטרים, תצוגות מתקדמות
**מטרה:** הפיכת המערכת לשימושית באמת ביום-יום.

1. תצוגת "ממתינים" — פילטר ייעודי במסך המשימות
2. קיבוץ לתיק — UI מלא (כפתור, דיאלוג, תצוגה מקובצת)
3. פילטרים מתקדמים במסך משימות (קטגוריה, טווח תאריכים, חיפוש טקסט)
4. תצוגת "מקובץ לפי לקוח" / "מקובץ לפי קטגוריה"
5. Shortcuts — מקלדת: `N` משימה חדשה, `D` סגירת משימה, `B` שינוי כדור
6. סימני מצב ויזואליים: צבעי דחיפות, badge של דד-ליין עובר

## 6. מחוץ לטווח (לשלבים הבאים)

דברים שעלו בדיון אך לא נכללים בשלב 1:

- ❌ **משימות חוזרות אוטומטית** (מע"מ דו-חודשי, ניכויים חודשי) — נוסיף כשתהיה ברורה הפעילות
- ❌ **לוג תקשורת** (Communication) — מה דיברנו, מה שלחנו — דורש משמעת תיעוד, נשקול בשלב ב׳
- ❌ **חברות כלקוחות** — דורש שדות ומסכים נפרדים, נעשה כשיהיה לקוח חברה ראשון
- ❌ **מעקב תיקים ברשויות** (מספרי תיקים במ"ה/מע"מ/ניכויים) — נוסיף כשיבוא
- ❌ **שכר טרחה וחיוב** — לא חלק מהמערכת כרגע
- ❌ **חיבור לדאטה בייס / cloud sync** — נשאר localStorage בשלב הנוכחי
- ❌ **אוטומציה** (תזכורות אוטומטיות, שליחת מיילים) — שלב ג׳

## 7. טרמינולוגיה (לשמירה על שפה אחידה)

| מונח עברי | מונח טכני | הערה |
|---|---|---|
| לקוח | Client | כולל עצמאים; חברות בעתיד |
| איש קשר | Contact | כל בן אדם שמופיע בתיק |
| משימה | Task | יחידת עבודה בודדת |
| תיק (עבודה) | Case | קיבוץ אופציונלי של משימות |
| כדור | ballWith | אצל מי המשימה ממתינה |
| קטגוריה | category | נושא העבודה (מ"ה/מע"מ/וכו׳) |
| תפקיד | role | תפקיד איש הקשר מול לקוח ספציפי |
| על השולחן | MyDesk | המסך הראשי — מה לעשות עכשיו |
